## Intro

很多现实中的问题，其计算逻辑远远超出人类能够理解的范围。例如语音识别等等。

为了解决这类问题，我们不再尝试直接写出计算逻辑，而是写出计算逻辑的一个范式，这个范式中包含很多参数，通过数据训练程序调整参数，最终达到满意的效果。

这个过程中，有如下几个要素是重要的：数据、模型、算法、目标函数。

### 数据

由样本组成，一般要满足独立同分布（independently and identically distributed），这里，独立的意思是上一个样本不会影响下一个样本，同分布的意思是它们遵循同样的规律（比如，如果训练集都是白天拍摄的街道照片，测试集都是晚上拍的，那就不太是同分布）。

所谓同分布，其实是在强调训练集要具有全面性。我们训练模型的目标，就是要让它能从变化的环境中识别不变的东西（比方说，识别不同角度、天气下拍摄的小猫照片），数据不可能完全遵循同样的规律。如果你只给模型一张照片，它只能识别跟训练集完全一样的照片；可随着你给的照片数量变多、覆盖情况更全面，它就能发现其中真正重要的部分。而之前街道的例子之所以会出现问题，是因为它的数据集不全面，模型认为你想让它学“白天的街道”而非是“街道”。

此外，传统机器学习通常不能直接处理变长数据，必须先转化为固定长度；深度学习对变长数据处理更自然。
### 模型

最后训出来的就叫模型。

### 目标函数

loss function，衡量模型输出好坏的。

为了避免过拟合问题，通常把数据集分成训练集和测试集。

过拟合是个很深奥的问题。因为模型的任务本质上是分成在训练集上优化，以及在测试集上测试泛化结果两步。如果在训练集上做得太好了，反而会泛化结果很差。

我们现在做的工作，显式看来都是在优化，其实是在隐式做泛化。所以 loss 降下来还远远不够。

### 算法

这里的算法不是模型的架构那个算法，这里算法说的是如何从数据中学习能最小化 loss function 的那个算法。

通常用的都是梯度下降，哪怕是训最新的模型用的也只是优化变种。

简单学一下梯度下降。

分成前向传播和反向传播两步。前向传播就是让模型跑一下算最终的 loss。反向传播就是倒过来，一层层地利用链式法则和前向传播的中间结果来计算梯度，最后根据学习率（步长）往梯度下降的方向走一步。

不过这样不是很容易落在局部最优解上吗？这个方法想要生效也许对函数的性质有一定要求，比方说首先得可导吧。

为什么不使用爬山算法或者模拟退火呢？

哦查了一下我知道了。

首先第一个，对函数确实有要求，比方说至少在绝大多数点可导。但是对于那些不太好的函数，比如 argmax，我们可以用类似的 softmax 来替代。所以可导的要求是比较好满足的。

第二个，爬山或者模拟退火虽然能克服局部最优解，但是它们没有利用函数可导的性质，相当于只是在使用零阶性质，效率太低。而我们最终的 loss function 其实最大的问题不是局部最优解，而是鞍点、平台处算得太慢。

总之模拟退火也不是完全没用，它在有很多不太好的局部最优解，或者拿不到梯度的时候是可以的，只不过我们目前解决的问题并不是这样的。

## 机器学习问题分类

d2l 上的分类比较烂，我们自己分一下。

### 学习方式

监督学习（有标准答案）、无监督学习（没有标准答案）、半监督学习（有的数据有答案）、自监督学习、强化学习（没答案、给奖励）

从这个角度来看，高中以前的学习是监督学习，此后的学习更像是强化学习。

自监督学习指的是，我的原始数据没有答案，但是我可以遮住一部分（比如句子里的一个词），然后让模型去猜，猜完之后跟原句比对，相当于自己抠出来一个答案。那现在 LLM 训练的方法都是这样的。

这里 d2l 还分出一个与环境互动来。确实，之前我们学习的行为都不会影响之后的结果（相当于之前说的独立），但是现在我们做出的决策会改变之后的数据，所以我们需要把自己之前的行为也纳入学习过程中。这个有点像 dp 算法，如果少考虑了一个因素引起了后效性，我们就把它加进来就好了。

包括一些序列学习，因为前面的输入会影响后面输入的分布，所以我们也需要学习这种内部结构。

### 输出形式

分类（输出离散）、回归（输出连续）、排序（输出有序）、生成。

### 数据类型

文本、图象、音频、视频、图。

### 任务目标

预测、描述（理解）、决策、生成、分类、异常检测。

## 历史

挺有意思的。

最开始深度学习没有得到发展，是因为算力不足。人们对智能的探索局限在一些有明显理论支撑、结果也具有高度可确定性的“确定”算法里，比如决策树等等。

但是之后，算力增长的速度超过了数据增长的速度，我们有了算力冗余，为了利用这部分算力，之前一些没被重视的算法重新被发现，深度学习开始成为主流。

这里列举了很多重要的工作，我应该之后认真阅读一下，起码搞懂它们的基本思想以及作用。

## 特点

深度学习是多层表示学习，越靠近输入的层次学到的是浅层具体的东西，越靠近输出的层次学到的是抽象的东西。

我感觉理解或者说表示这个任务应该是最基本的任务，可是还有哪些任务类型呢？生成。还有别的吗？这些任务之间是否存在一个先后关系？

传统机器学习把特征工程跟学习分开了，这不好。深度学习把它们统一成一个系统，虽然复杂度上升了，可解释性下降了，但是能力和泛用性变强了。系统是对的。

之前深度学习取得的进展，很大一部分归功于廉价传感器、互联网带来的数据增长。未来会依赖于什么？
